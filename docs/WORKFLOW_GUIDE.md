## ğŸ“ Admin Workflow Tips

### Efficient Review Process

1. **Set High Target (100+)**: Ensures long review sessions without interruption
2. **Review in Batches**: Accept/reject multiple questions per session
3. **Check Similarity**: Use similar questions endpoint to avoid duplicates
4. **Trust Auto-Gen**: System maintains pool automatically

### Monitoring Pool Health

```bash
# Check current pool size
curl -H "Authorization: Bearer TOKEN" \
  "http://localhost:3333/api/admin/questions/review?stats=true"

# Response:
{
  "to_review": 98,
  "accepted": 250,
  "rejected": 15
}
```

### Adjusting Target Size

```bash
# Increase to 150 during high-traffic periods
curl -X POST -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"targetPoolSize": 150}' \
  "http://localhost:3333/api/admin/generation-settings"
```

- Set a reasonable target (50-200)
